Recently was working around DNS and thought to put it here! 

DNS acts as the phonebook of the internet üåê. It converts the web address to IP address so that we don't have to 
remember complicated IP addresses ü§©.
Typical DNS lookup goes like this :

1) Lets type example.com in a web browser. This request is first looked up in browser DNS cache, if not found then Step 2 

2) Then it looks into the system DNS cache. If not found then step 3 

3) Then it looks into `\etc\hosts`
The host file is a method of maintaining hostname to IP mapping from the ARPANET days. If not found then Step 4 

4) The system tries to connect to your configured DNS Servers. You can manually set your DNS Servers, or your connected networks can configure it for you.

5) Now the resolver looks into its DNS cache if not found then tries to connect to root nameserver (.) 
You can do `dig .` to find root nameserver your DNS server is trying to connect. At present, there are 13 
root nameservers named with the letters 'A' to 'M.' Example `a.root-servers.net.`

8) Resolver requests IP addresses to authoritative nameservers. Then returns the result to the system.

```
‚ûú dig @a.iana-servers.net. -t A example.com

; <<>> DiG 9.10.6 <<>> @a.iana-servers.net. -t A example.com
; (1 server found)
;; global options: +cmd
;; Got answer:
;; ->>HEADER<<- opcode: QUERY, status: NOERROR, id: 5682
;; flags: qr aa rd; QUERY: 1, ANSWER: 1, AUTHORITY: 0, ADDITIONAL: 1
;; WARNING: recursion requested but not available

;; OPT PSEUDOSECTION:
; EDNS: version: 0, flags:; udp: 4096
;; QUESTION SECTION:
;example.com.           IN  A

;; ANSWER SECTION:
example.com.        86400   IN  A   93.184.216.34

;; Query time: 281 msec
;; SERVER: 199.43.135.53#53(199.43.135.53)
;; WHEN: Wed May 06 22:58:40 IST 2020
;; MSG SIZE  rcvd: 56
```

Using the IP address `93.184.216.34`, the web browser connects to the host. 

Every stage maintains a cache for some number of seconds called `TTL` that returns as part of every query. In the following 
result, the TTL is `86400` seconds

```
example.com.        86400   IN  A   93.184.216.34
```

A resolver can cache the contents of the query for 86400 seconds. This caching helps to speed up the process and 
reduces the load on DNS servers.
